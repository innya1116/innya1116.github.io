프로젝트 목적

  • SK innovation ESG Data Platform 구축
  • PC 사이트 제작(1440px 해상도 이상을 기준으로 작업)
  • 국문, 영문 사이트 제작

프로젝트 환경

  • 문서 및 버전 : HTML5, CSS3
  • 인코딩 : utf-8
  • 브라우저 호환성 : Edge, Chrome, Firefox, Safari 최신 버전 (모바일 버전 없음)

HTML 작업 규칙

  • 모든 html 코드는 소문자로 작성한다.
  • 들여쓰기는 공백 2칸으로 사용한다.
  • 가독성을 고려하여 빈 줄을 추가할 수 있다. 단 연속적으로 1줄을 초과하지 않는다.
  • attribute="value" 작성 시, 반드시 큰따옴표("")를 사용한다.
    (예외로 사용자 정의 속성(data-)에 중복 옵션 설정을 위해 중괄호({})를 사용할 경우, 작은 따옴표('')를 사용할 수 있다.)
  • 자동닫기 요소의 마지막 /는 생략(예. img, input, br 등)
  • 차트의 경우 솔루션 차트를 개발에서 적용함(퍼블리싱에서는 차트가 들어갈 영역을 임시이미지로 표기 후 가이드 전달)

CSS 작업 규칙

Class

  • BEM 방법론을 따른다 (http://getbem.com/naming/)
  • 기본정의 : block요소name--Modifier(변형자 or 상태)__Element(자식요소. key와 value는 하이픈(-)사용)
  • --modifier를 사용할 경우 원형이 있는 상태에서 --modifier 형태의 클래스를 추가로 붙여준다. 
    <a class="btn btn--outlined" href="#">버튼입니다.</a>
  • .부모명__자식, .부모명__자식--확장, .부모명__자식명구분-자식명구분
    <div class="summary-info">
  <h2 class="summary-info__title"> ... </h2>
  <div class="summary-info__contents summary-info__contents--wide">
    ...
  </div>
</div>

속성 선언 순서

  1. display
  2. float
  3. position, top, left, right, bottom
  4. width, height
  5. margin, padding
  6. border
  7. background
  8. color, font, letter-spacing, white-space
  9. text-align, vertical-align, text-decoration, text-indent
  10. z-index
  11. 기타(위에 언급하지 않은 나머지 속성들)

영문 CSS

  • 영문은 국문 CSS를 베이스로 적용한다.
  • 별도의 스타일 적용이 필요할 경우 html[land='en']을 앞에 붙여 영문페이지에서만 적용되도록 작성한다.

주석 표기

영역주석

<div>
  <p>시작과 끝이 있고, 마크업의 영역을 표시한다.</p>
</div>
<!--// 영역명 -->

<div class="area">
  <p>class의 범위를 표기하는 주석, 영역이 끝나는 마지막 닫기 태그 옆에 표시한다.</p>
</div><!--// .area -->
