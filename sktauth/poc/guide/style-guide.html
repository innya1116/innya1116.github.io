Form

접근성
  • 접근성 준수를 위해 label을 사용하는 경우 input의 id와 label의 for 값을 반드시 매칭해야 함
  • label 없이 사용하는 경우 title을 이용하여 해당 input에 대한 설명 추가 필요

Input Default

label의 for 값과 input의 id 값 매칭 필요

<div class="inputbox">
	<div class="input">
		<label class="placeholder" for="auth-input-default">인증 input 기본형</label>
		<input type="text" class="authInput" id="auth-input-default">
	</div>
</div>

Number Input - 숫자만 입력되는 input

- 법정 생년월일 및 인증번호 등 숫자만 입력되는 input 사용 시, inpNumber 클래스 추가 필요

<div class="inputbox">
	<div class="input">
		<label class="placeholder" for="auth-number">숫자만 입력되는 input</label>
		<input type="text" class="authInput inpNumber" id="auth-number">
	</div>
</div>

Cell Number Input - 휴대폰 번호 입력 input

- 휴대폰 번호 입력 input 인 경우, inpCellNumber 클래스 추가시 자동으로 dash(-)가 추가됨

<div class="inputbox">
	<div class="input">
		<label class="placeholder" for="auth-cell-number">휴대폰 번호('-' 제외한 숫자만 입력)</label>

	</div>
</div>

Check Point - Default: Close

운영사이트의 checkpoint와 디자인이 달라져서 마크업이 변경되었고, 상세 내용의 마크업은 버튼을 제외하고는 기존 운영 코드와 동일하게 반영했습니다.

CHECK POINT
    • 인증문자는 114 번호로 발신되오니 해당 번호의 단말기 스팸설정 여부 및 스팸편지함을 확인해 주세요.
    • 스팸필터링 부가서비스 가입 후 114 번호를 스팸설정 하신 경우 스팸설정을 해제해 주세요.
    • 착신 관련 서비스에 가입되어 있는 경우 SMS 인증문자가 수신되지 않을 수 있습니다.
    • 현재 사용중인 휴대폰이 USIM 단독 상태이거나 휴대폰과 USIM카드가 분리된 상태에서는 SMS 인증을 받으실 수 없습니다.

Html Call Function(js)

Function - mvJs.utils.sendFrmSms.CheckBtnAuth();

  • 정보입력 페이지에서 다음 스텝으로 진행되는 버튼 활성화 함수(필요에 따라 호출)
  • 현재는 PASS인증서 정보입력 페이지에만 적용되어 있음
  • .inPop > input 모두 입력 되었을 경우 .popup.authPopup .rArea > button 버튼 활성화
  • input[type=text] : 한글 자 이상 입력 되어 있을 경우
  • input[type=radio], input[type=checkbox] : 체크 되었을 경우, 알뜰폰 체크 박스 제외
